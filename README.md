---

## Опис реалізації

### 1. Ієрархія компонентів тексту

* **`Text`**: Найвищий рівень, що розбиває вхідний рядок на об'єкти `Sentence` та розділові знаки (крапки, знаки питання тощо).
* **`Sentence`**: Колекція об'єктів `Word` та `Punctuation`. Реалізує інтерфейс `Comparable` для сортування речень за першим словом.
* **`Word`**: Складається зі списку об'єктів `Letter`. Дозволяє працювати зі словом як з окремою сутністю.
* **`Letter`**: Представляє окремий символ тексту.
* **`Punctuation`**: Зберігає розділові знаки та пробіли.

### 2. Технічні особливості

* **`TextElement`**: Спільний інтерфейс для всіх складових частин тексту (`Word`, `Sentence`, `Punctuation`), що дозволяє використовувати поліморфізм при обробці.
* **Регулярні вирази**: Використання класів `Pattern` та `Matcher` для точного розбиття тексту на компоненти, враховуючи специфіку мови та знаків.
* **Lombok**: Використання анотацій `@Getter` та `@NonNull` для зменшення кількості шаблонного коду.

---

## Сортування та пошук

* **Сортування речень**: У класі `Sentence` реалізовано метод `compareTo`, який порівнює речення за лексикографічним порядком їхніх перших слів.
* **Порівняння об'єктів**: У всіх класах (`Letter`, `Word`, `Sentence`) перевизначено метод `equals()`, що дозволяє здійснювати пошук ідентичних елементів у колекціях.
